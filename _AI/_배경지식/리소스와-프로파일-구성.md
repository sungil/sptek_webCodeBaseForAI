# 리소스와 프로파일 구성

이 문서는 `src/main/resources` 하위의 리소스 위치와 profile별 설정 구조를 확인할 때 본다.
Spring profile의 일반 사용법이 아니라, 이 저장소의 리소스 배치 기준만 다룬다.

## 기본 구조

프로파일은 `local`, `dev`, `stg`, `prd`를 기준으로 나뉜다.
예시 업무 프로젝트는 `src/main/resources/_sptek/_webFrameworkExample/application-{profile}.yml`에서 기능별 설정 파일을 `spring.config.import`로 가져온다.

리소스도 Java 코드 패키지와 같은 기준으로 분리한다.
실제 업무 프로젝트 리소스는 `src/main/resources/{companyName}` 기준 하위 경로를 사용한다.
개별 업무 도메인 리소스는 `{companyName}/_{domainName}` 하위에 두고, 여러 업무 도메인이 공유하는 리소스는 `{companyName}/__{companyName}Common` 하위에 둔다.
예시 프로젝트 리소스와 실제 업무 리소스가 섞이지 않도록 확인한다.

## 변경 기준

profile별 설정 키를 추가하거나 구조를 바꿀 때는 `local/dev/stg/prd` 파일을 함께 검토한다.
값이 환경마다 달라야 하면 구조만 맞추고 실제 운영값은 추측하지 않는다.

기능별 설정 파일이 profile별로 분리되어 있으면 `application-{profile}.yml`의 import 경로까지 함께 확인한다.
