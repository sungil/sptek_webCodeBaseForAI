# 지침 충돌 처리 정책

이 문서는 `_AI/` 하위 문서들 사이의 내용이 서로 다르거나 충돌해 보일 때 처리 기준을 정한다.
AI Agent는 문서 간 내용이 다르게 보인다는 이유만으로 임의로 한쪽을 무시하지 않고, 먼저 적용 범위와 현재 작업기준과의 관련성을 확인한다.


## 지침 충돌로 보지 않는 경우

다음은 지침 충돌로 보지 않는다.
- 공통 문서가 기본 기준을 정하고, 영역별 문서가 해당 영역의 세부 기준을 추가하는 경우
- 작업정책 문서가 원칙을 정하고, 작업수행절차 문서가 수행 순서를 구체화하는 경우
- 배경지식 문서가 판단에 필요한 설명을 제공하고, 정책 문서가 실제 준수 기준을 정하는 경우
- 같은 기준을 서로 다른 표현으로 설명하지만 실제 작업 결과가 달라지지 않는 경우


## 판단 기준

지침 충돌 가능성이 있으면 아래 기준으로 판단한다.
- 역할 확인, 작업범위, 민감정보 보호, Git 작업, 보안, 운영 설정값 보호처럼 안전과 권한에 관련된 내용은 더 보수적인 기준을 따른다.
- 작업 대상 업무영역 내부의 패키지 구조, 네이밍, 데이터 접근 방식, 업무 규칙 배치처럼 해당 영역의 구현 방식에 관한 내용은 현재 작업기준에 해당하는 영역 문서를 우선 검토한다.
- 작업정책 문서와 작업수행절차 문서가 다르게 보이면 작업정책 문서를 먼저 기준으로 삼고, 절차 문서는 정책을 위반하지 않는 범위에서 따른다.
- 배경지식 문서는 정책이나 절차를 대체하지 않는다. 배경지식과 정책이 다르면 정책을 따른다.
- 어느 기준을 따라야 할지 명확하지 않으면 작업을 진행하지 않고 사용자에게 확인한다.


## 보고 기준

지침 충돌 가능성이 있는 문서를 발견하면 작업 전 안내 또는 최종 답변에 다음 내용을 포함한다.
- 내용이 다르거나 충돌해 보이는 문서 경로
- 서로 다르게 보이는 내용
- 현재 작업에서 적용하려는 기준
- 사용자 확인이 필요한 이유
