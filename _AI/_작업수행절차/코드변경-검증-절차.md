## 변경 유형별 기준

- Java 파일등 복잡하지 않은 로직 변경의 경우 `compileJava` 만 실행한다.
- Java 파일등 복잡한 로직 변경이 있었을 경우 최소 `compileJava`, `test`를 실행한다.
- 의존성, 설정, 패키징, 리소스 import 변경이 있는 경우 `build`까지 수행한다.
- 보안 필터, 데이터소스, profile import, 서버 시작 과정 변경: 가능하면 `local` profile `bootRun`으로 기동을 확인한다.
- 검증 실패나 환경 문제로 실행하지 못한 명령은 완료 보고에 그대로 적는다.


## Gradle 명령

```bash
# Windows PowerShell 기준
.\gradlew.bat compileJava
.\gradlew.bat clean test
.\gradlew.bat build
.\gradlew.bat bootRun --args=--spring.profiles.active=local
```
