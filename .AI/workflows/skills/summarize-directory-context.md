# Summarize Directory Context Workflow

이 문서는 특정 디렉토리의 역할을 요약하는 하위 `AI.md`를 생성하거나 갱신할 때 따르는 절차다. 루트 `AI.md`의 공통 원칙과 충돌하지 않으며, 하위 문서는 해당 디렉토리 범위에서 더 구체적인 작업 맥락을 제공한다.

## 목표

- 디렉토리 안의 파일들이 각각 어떤 역할을 하는지 요약한다.
- 앞으로 비슷한 역할의 파일이나 클래스를 만들 때 이 디렉토리에 두는 것이 적합한지 판단할 수 있게 한다.
- Java 패키지 디렉토리라면 어떤 기능군의 클래스들이 모여 있는지 밝히고, controller/service/repository/dto/entity/config/filter/interceptor/aspect/advice 등 계층과 역할을 구분한다.
- 상세 구현 설명보다 AI 코딩 시 참조할 위치 판단, 수정 주의사항, 관련 파일 탐색 기준을 우선한다.

## 사용 시점

- 사용자가 특정 디렉토리에 `AI.md` 요약 파일 생성을 요청한 경우.
- 사용자가 기존 디렉토리 `AI.md` 업데이트를 명시적으로 요청한 경우.
- 커밋 대상 파일의 추가, 삭제, 이동, 역할 변경이 기존 디렉토리 요약과 맞지 않게 된 경우.
- 새 파일이나 새 클래스를 어디에 둘지 판단하기 위해 디렉토리 역할을 문서화해야 하는 경우.

## 분석 범위

1. 대상 디렉토리의 직접 파일과 직접 하위 디렉토리를 먼저 확인한다.
2. Java 패키지 디렉토리는 클래스명, 애노테이션, 상속/구현, Bean 등록 방식, 주변 패키지 관계를 확인한다.
3. 리소스 디렉토리는 설정 파일, mapper, template, static asset, SQL, i18n 등 파일 유형별 역할을 구분한다.
4. HTTP Client 디렉토리는 요청 대상 URL, 인증 흐름, 환경 변수 사용, 수동 검증 목적을 확인한다.
5. 기존 하위 `AI.md`가 있으면 그 범위는 상세 분석 대상으로 중복 확장하지 말고, 현재 디렉토리의 자식 영역으로 요약한다.
6. 민감 파일은 루트 `AI.md`의 제한을 따른다. private env, keystore, 운영 비밀값은 내용을 출력하거나 요약에 노출하지 않는다.

## 작성 형식

하위 `AI.md`는 다음 구조를 기본으로 사용한다. 필요 없는 섹션은 생략할 수 있지만, `디렉토리 역할`과 `작업 기준`은 가능한 유지한다.

```md
# AI.md

## 디렉토리 역할

이 디렉토리는 ...

## 파일과 하위 디렉토리

- `파일명`: ...
- `하위디렉토리/`: ...

## 작업 기준

- ... 성격의 파일은 이 디렉토리에 둔다.
- ... 성격의 파일은 다른 경로를 먼저 검토한다.

## 주의사항

- ...

## 관련 참조

- `상대/경로`
```

## 작성 기준

- “무엇을 하는 파일들이 모이는 곳인가”를 먼저 쓴다.
- “앞으로 어떤 파일을 여기에 추가하는 것이 자연스러운가”를 명확히 쓴다.
- 파일별 설명은 한 줄 중심으로 쓰되, 디렉토리 판단에 필요한 정보는 빠뜨리지 않는다.
- 구현 세부 로직을 길게 풀지 말고, 관련 코드를 찾기 위한 키워드와 연결 지점을 남긴다.
- 루트 `AI.md`와 같은 공통 규칙을 반복하지 않는다. 하위 디렉토리에만 적용되는 구체 사항을 쓴다.
- 추측으로 역할을 단정하지 않는다. 이름만으로 부족하면 파일 내용을 열어 확인한다.
- 생성 파일, 로그, DB 파일, 빌드 산출물은 일반적으로 문서화 대상에서 제외한다. 단, 디렉토리 역할 설명에 필요한 경우 “생성물 위치” 정도로만 언급한다.
- 기존 `AI.md`가 있으면 전체를 덮어쓰기 전에 현재 내용과 새 분석 결과를 병합한다.

## 커밋 전 갱신 기준

커밋 요청을 받았을 때 다음 조건 중 하나에 해당하면 관련 디렉토리의 `AI.md`를 확인하고 필요 시 갱신한다.

- 새 파일이나 새 클래스가 추가되어 디렉토리의 “파일과 하위 디렉토리” 목록에 반영할 필요가 있다.
- 기존 파일의 역할이 바뀌어 설명이 틀려졌다.
- 파일이 이동되거나 삭제되어 “이곳에 어떤 파일을 둔다”는 기준이 달라졌다.
- 새 하위 디렉토리가 생겨 디렉토리 구조 판단에 영향을 준다.
- 코드 패키지의 기능군이 바뀌어 새 클래스 작성 위치 안내가 달라졌다.

단순 내부 구현 변경으로 디렉토리 역할, 파일 역할, 배치 기준이 바뀌지 않았다면 `AI.md`를 억지로 수정하지 않는다.
