# 사용자역할과 작업영역 확인 정책

이 문서는 새로운 세션이 시작될 때 AI Agent가 사용자와의 대화를 진행하거나 작업을 수행하기 전에 반드시 확인하고 적용해야 하는 최우선 지침이다.

## 기본 원칙

새 세션에서 개발 작업을 시작하기 전에는 다른 작업보다 먼저 현재 사용자의 역할과 작업 영역을 확인한다.
사용자의 역할과 작업 영역이 확인되지 않은 상태에서는 업무 가능 상태만 답변하고 코드 분석, 파일 조회, 구현, 검증, 완료 보고를 시작하지 않는다.

역할과 작업 영역은 일반 텍스트 목록으로만 보여주지 않는다.
사용자가 화살표 키 또는 선택 UI로 고를 수 있는 도구가 제공되는 환경이라면 반드시 그 선택 UI를 사용한다.
선택 UI를 제공할 수 없는 실행 환경에서만 예외적으로 텍스트 질문을 사용할 수 있다.
텍스트 질문을 사용하는 경우에는 선택지에 번호를 붙이고, 사용자가 번호로 답할 수 있게 안내한다.
역할을 먼저 확인한 뒤, 이어서 작업 영역을 확인한다.

역할은 다음 중 하나로 확인한다.

- 비즈영역 설계
- 비즈영역 기획
- 비즈영역 개발
- Framework 개발

작업 영역은 추상 텍스트로 작성하지 않는다.
AI Agent는 현재 저장소의 실제 패키지 구조를 먼저 조사한 뒤 다음 규칙에 맞는 실제 작업 영역 목록을 생성해서 선택 UI로 제공한다.

- Framework코드영역: `src/main/java/com/_sptek/__webFramework`
- 회사 공통 영역: `src/main/java/com/{companyName}/__{companyName}Common`
- 업무 도메인 영역: `src/main/java/com/{companyName}/_{domainName}`

예를 들어 현재 저장소에서 `com.cesco._marketing`, `com.cesco._sales`가 존재하면
작업 영역 선택지에는 `com.{companyName}._{domainName}` 같은 추상 문구가 아니라
`com.cesco._marketing`, `com.cesco._sales`를 실제 이름으로 표시한다.

텍스트 질문을 사용해야 하는 경우에는 다음 형식을 따른다.

```text
역할 번호를 입력해 주세요.
1. 비즈영역 설계
2. 비즈영역 기획
3. 비즈영역 개발
4. Framework 개발
```

역할을 확인한 뒤에는 현재 저장소에서 조사한 실제 작업 영역 목록을 같은 번호 입력 형식으로 제시한다.

## 역할별 작업 기준

사용자의 역할이 `Framework 개발`이면 Framework코드영역을 포함해 전체 영역을 자유롭게 조사하고 수정할 수 있다.

사용자의 역할이 `비즈영역 개발`이면 선택된 작업 영역 내부에서 개발 작업을 수행하는 것을 기본으로 한다.
선택된 영역 밖의 Framework코드영역, 공통 영역, 다른 업무 도메인 영역을 수정해야 하는 경우에는 수정 사유와 영향 범위를 먼저 설명하고 사용자의 명시적인 확인을 받은 뒤 작업한다.

사용자의 역할이 `비즈영역 설계` 또는 `비즈영역 기획`이면 질문에 답하고 구조, 정책, 설계 방향을 제안할 수는 있지만 실제 코드는 수정하지 않는다.
문서 수정도 사용자가 명시적으로 요청한 경우에만 수행한다.

## 참고 순서

개발 작업에 필요한 구현 방식과 기준은 다음 순서로 확인한다.

1. Framework코드영역
2. `_webFrameworkExample` 예시 프로젝트 영역
3. 사용자가 선택한 작업 영역
4. 위 영역에서 명확한 참고 정보를 찾을 수 없는 경우에만 다른 업무 도메인 영역

다른 업무 도메인의 코드는 참고 목적으로만 확인한다.
사용자의 확인 없이 다른 업무 도메인 영역을 수정하지 않는다.

참고한 구현이 현재 작업 영역의 구조, 정책, 도메인 특성과 충돌하는 경우에는 그대로 복사하지 않고 현재 작업 영역의 기준을 우선한다.
