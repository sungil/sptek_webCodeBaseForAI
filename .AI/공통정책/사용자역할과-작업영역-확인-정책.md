# 사용자 역할과 작업 영역 확인 정책

이 문서는 새 세션에서 AI Agent가 모든 대화와 작업에 앞서 반드시 적용해야 하는 최우선 지침이다.

## 기본 원칙

새 세션이 시작되면 먼저 사용자의 역할과 작업 영역을 확인한다.

확인이 완료될 때까지 역할과 작업 영역을 확인하기 위한 질문 외에는 답변, 조사, 분석, 코드·문서 수정, 명령 실행 등 어떠한 작업도 진행하지 않는다.

역할과 작업 영역은 선택 UI로 확인한다. 선택 UI를 사용할 수 없는 경우에만 번호가 있는 텍스트 목록을 제공한다.

## 역할 선택

사용자 역할은 다음 중 하나로 확인한다.

- 비즈영역 설계
- 비즈영역 기획
- 비즈영역 개발
- Framework 개발

`Framework 개발`을 선택하면 작업 영역을 별도로 확인하지 않고 다음과 같이 안내한다.

```text
Framework 개발은 모든 영역에서 조사와 작업이 가능합니다.
별도의 작업 영역을 선택하지 않습니다.
```

## 작업 영역 선택

`Framework 개발` 이외의 역할만 작업 영역을 선택한다.

현재 저장소의 실제 패키지 구조를 조사하여 다음 영역을 선택지로 제공한다.

- 회사 공통 영역: `com.{companyName}.__{companyName}Common`
- 업무 도메인 영역: `com.{companyName}._{domainName}`

Framework 영역은 선택지에서 제외하며, 추상적인 예시 대신 실제 패키지명을 표시한다.

선택지의 마지막에는 항상 다음 항목을 추가한다.

- 새 작업영역 추가

`새 작업영역 추가`를 선택하면 아직 업무 영역을 생성하지 않고, 다음 문구로 신규 업무 영역 식별자만 먼저 입력받는다.

```text
companyName.domainName 형식으로 업무 영역을 입력해 주세요.
```

입력값을 받은 뒤에는 `.AI/작업-수행절차/신규-업무영역-추가-절차.md`를 읽고 그 지침에 따라 다음 단계를 진행한다.

## 역할별 작업 범위

`Framework 개발`은 Framework, 회사 공통 및 모든 업무 도메인 영역을 자유롭게 조사하고 수정할 수 있다.

`비즈영역 개발`은 선택한 업무 도메인과 같은 회사의 공통 영역을 수정할 수 있다. 회사 공통 영역을 선택한 경우에는 선택한 회사 공통 영역을 수정할 수 있다. Framework 또는 다른 도메인 수정은 사유와 영향 범위를 설명하고 사용자 확인 후 수행한다.

`비즈영역 설계` 및 `비즈영역 기획`은 분석, 설계 및 제안만 수행하며 코드는 수정하지 않는다. 문서는 사용자가 명시적으로 요청한 경우에만 수정한다.

## 참고 순서

개발 작업은 다음 순서로 기존 구현을 참고한다.

1. Framework 영역
2. `_webFrameworkExample`
3. 같은 회사의 공통 영역
4. 선택한 업무 도메인
5. 필요한 경우에만 다른 업무 도메인

다른 도메인은 참고할 수 있지만, `Framework 개발` 역할이 아닌 경우 사용자 확인 없이 수정하지 않는다.
