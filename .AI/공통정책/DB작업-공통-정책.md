# DB작업 공통 정책

이 문서는 DB 제품 종류와 관계없이 DB 작업을 할 때 지켜야 하는 공통 기준을 정의한다.

- 데이터 삭제, 초기화, 스키마 변경, 복제 구성 변경은 실행 전에 대상과 영향을 명확히 확인한다.
- 대량의 데이터 삭제·변경 또는 영향 범위가 넓은 스키마 변경은 대상, 예상 영향 및 복구 가능 여부를 사용자에게 설명한 후 명시적인 재확인을 받아 수행한다.
- DB 작업 전에 실행 애플리케이션에서 활성화된 datasource 설정을 먼저 확인한다.
- 활성화된 datasource를 확인할 수 없는 경우에는 `@Enable_DatasourceOfH2_At_Main`을 활성화해 H2 DB 기준으로 작업을 진행하겠다고 사용자에게 먼저 안내한다.
- DB 작업에 사용하는 쿼리는 표준 SQL을 우선으로 작성한다.
- 특정 DBMS에 종속된 문법, 함수 또는 기능은 사용자가 명시적으로 요청하거나, 표준 SQL만으로 요구사항을 충족하기 어려운 경우에 한해 사용한다.
- 테이블명, 컬럼명, PK, FK, index, 데이터 타입, 길이 및 제약조건을 정의할 때 표준 메타정보가 있으면 해당 기준을 따른다.
- 명확한 기준이 없는 경우에는 기존 테이블에서 동일하거나 유사한 역할을 수행하는 항목을 확인하여 명명 방식과 데이터 구조의 일관성을 유지한다.
- 참고할 기존 항목도 없는 경우에는 데이터의 의미, 예상 범위 및 사용 목적을 고려하여 적절한 이름, 타입, 크기 및 제약조건을 산정한다.
