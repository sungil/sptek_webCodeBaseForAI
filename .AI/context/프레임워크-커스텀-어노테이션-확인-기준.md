이 문서는 `__webFramework`의 커스텀 애노테이션을 새 기능 개발이나 분석 전에 어떻게 확인할지 안내한다.

## 기본 개념

`Enable_*` 애노테이션은 프레임워크 기능을 명시적으로 켜거나, 특정 Controller/메서드/파라미터/필드에 프레임워크 처리를 적용하기 위한
표시다. 어노테이션 이름을 통해 해당 어노테이션의 기능을 유추할 수 있다.

## 확인 순서

- 새 기능을 만들거나 기존 기능을 분석할 때는 먼저 현재 코드에서 관련 애노테이션을 검색한다.
- 애노테이션 파일만 보고 동작을 판단하지 않는다. 실제 동작은 그 애노테이션을 읽는 코드에서 결정된다.
- 함께 확인할 것:
    - 애노테이션 JavaDoc
    - `@Target`
    - `@HasAnnotationOnMain_At_Bean`
    - `MainClassAnnotationRegister`
    - Filter, Interceptor, Aspect, ControllerAdvice, ArgumentResolver 연결 코드
    - `_webFrameworkExample`의 사용 예시
    - 관련 profile 설정과 테스트

## 주의점

- `At_Main` 애노테이션은 전역 기능 스위치다. 업무 코드에서 단순 feature flag처럼 남용하지 않는다.
- Controller, 메서드, 파라미터, 필드에 붙는 애노테이션은 선택적으로 프레임워크 처리를 적용한다는 표시다. 실제 처리 방식은 연결된
  resolver, interceptor, aspect, message converter를 확인해야 한다.
- `String value()`가 있는 애노테이션은 기능 옵션인지, 로그/태그용 값인지 사용처를 먼저 확인한다.
