

## 상세 확인 기준

`__webFramework` 내부의 상세 책임은 이 문서에 고정 목록으로 길게 유지하지 않는다. 기능 리팩토링으로 패키지와 클래스명이 계속 조정될 수 있으므로,
먼저 디렉토리명과 파일명을 단서로 삼아 현재 구조를 확인한다.

상세 내용을 파악할 때는 다음 순서로 본다.

1. 대상 기능과 가까운 디렉토리명
2. 클래스/파일명과 JavaDoc
3. 관련 `@Enable_*` 애노테이션
4. Bean 등록 조건, Filter/Interceptor/Aspect 연결 지점
5. `_webFrameworkExample`의 사용 예시
6. 테스트 코드와 profile 설정

## 사용 기준

- 코드를 새로 만들기 전에 현재 `__webFramework` 안에 같은 목적의 기능이 있는지 검색한다.
- 프레임워크 코드만으로 처리 가능한 기능은 중복 구현하지 않는다.
