# .AI 디렉토리 안내

`.AI/`는 특정 Agent 도구에 종속되지 않는 공통 작업 기준과 보조 자료를 둔다.
루트 `AI.md`를 먼저 읽고, 여기서는 현재 작업에 필요한 문서만 골라서 확인한다.

## 디렉토리 역할

- `프로젝트-이해자료/`: 이 저장소의 코드 영역 구분, 프레임워크 확장 방식, 리소스/profile 구조, 예시 프로젝트 참고 기준을 설명한다.
- `공통정책/`: 작업 종류와 관계없이 지켜야 하는 보존, 보안, Git, DB 공통/H2 DB, 신규 코드 작성, Framework코드영역/업무코드영역 코딩 스타일, 업무코드영역 패키지 구성 기준을 둔다.
- `작업-수행절차/`: 검증 명령, 커밋 메시지, 코드 주석, 읽기전용 DB 조회, AI 컨텍스트 내보내기처럼 별도 절차가 필요한 작업을 설명한다.
- `_ai-generated/`: AI가 작업 중 재사용하는 스크립트, SQL 조각, 보조 산출물을 둔다.

## 읽는 기준

모든 문서를 한 번에 읽지 않는다.
파일명과 디렉토리명을 보고 현재 요청과 직접 관련된 문서만 선택한다.

예를 들어 코드 위치가 문제라면 `프로젝트-코드영역-구분.md`를 보고, `@Enable_*` 어노테이션이나 filter/interceptor 연결이 문제라면 `프레임워크-확장방식-확인기준.md`를 본다.
profile 설정이나 리소스 위치가 관련되면 `리소스와-프로파일-구성.md`를 확인한다.
업무코드영역을 새로 개발한다면 `업무코드영역-코딩스타일-정책.md`를 보고, 새 패키지 구조가 필요하면 `업무코드영역-패키지구성-정책.md`를 함께 확인한다.
DB 접근 방식으로 MyBatis/JPA 선택이나 hybrid 사용이 필요하면 `업무코드영역-데이터접근기술-선택-정책.md`를 확인한다.
Framework코드영역을 수정한다면 `Framework코드영역-코딩스타일-정책.md`를 확인한다.

## 작성 기준

문서 하나에는 하나의 목적만 담는다.
AI가 일반적으로 처리할 수 있는 검색 방법, Spring 기본 사용법, 과도하게 세부적인 현재 파일 목록은 별도 지침으로 만들지 않는다.

프로젝트 구조가 바뀌면 파일명과 본문을 함께 갱신한다.
파일명만 보고도 읽어야 할지 판단할 수 있도록 구체적인 한글 이름을 사용한다.
