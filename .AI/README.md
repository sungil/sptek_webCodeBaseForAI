# .AI 디렉토리 안내

`.AI/`는 Codex, Claude, GitHub Copilot 등 특정 AI Agent 도구에 종속되지 않는 공통 작업 기준과 프로젝트 참고 자료를 관리하는 디렉터리다.
AI Agent는 새 세션이 시작되면 역할과 업무범위를 먼저 확인하고, 이후 현재 작업에 필요한 문서만 이 디렉터리에서 선별하여 읽는다.

## 내부 디렉토리 역할 기준

- `배경지식/`: 현재 저장소의 구조, 코드 영역 구분, Framework 확장 방식, 리소스 구성 등 프로젝트를 이해하는 데 필요한 자료를 둔다.
- `정책/`: 작업 종류와 관계없이 항상 적용해야 하는 보존, 보안, Git, DB, 코딩 스타일 및 패키지 구성 기준을 둔다.
- `작업-수행절차/`: 검증, 커밋, 주석, DB 조회, 작업 보고 등 특정 작업을 수행할 때 따라야 하는 순서와 방법을 둔다.
- `_ai-generated/`: AI Agent가 작업 중 생성하거나 재사용하는 스크립트, SQL 조각 및 보조 산출물을 둔다.

위 각 주요 디렉터리 바로 아래의 Markdown 파일은 모든 작업영역에 공통으로 적용하거나 참고한다.
위 각 주요 디렉터리 하위의 `{companyName}/__{companyName}Common/` 경로는 해당 회사 공통 영역 전용 문서 공간이다.
위 각 주요 디렉터리 하위의 `{companyName}/_{domainName}/` 경로는 해당 업무 도메인 전용 문서 공간이다.
영역 전용 Markdown 파일은 해당 회사 공통 영역 또는 업무 도메인을 조사하거나 수정할 때만 추가로 적용하거나 참고한다.
예를 들어 `com.{예시회사명}._{예시업무A}` 작업에서는 `.AI/정책/{예시회사명}/_{예시업무A}/` 하위 문서를 추가로 확인하고, `com.{예시회사명}._{예시업무B}` 작업에는 적용하지 않는다.
같은 회사의 업무 도메인 작업 중 회사 공통 영역도 함께 조사하거나 수정해야 하면, 해당 회사 공통 영역의 문서도 함께 확인한다.

## 필요한 문서 찾기

작업에 필요한 정보나 지침을 확인하기 위해 모든 문서를 한꺼번에 읽지 않는다.
먼저 디렉터리명과 파일명을 확인하여 현재 작업과 관련 가능성이 높은 문서를 선별한다. 이후 작업과의 관련성을 판단하면서 읽을 문서의 순서와 범위를 단계적으로 좁혀간다.
파일명만으로 관련 문서를 판단하기 어려운 경우에만 상위 디렉터리의 안내 문서나 인접 문서를 추가로 확인한다.
특정 업무 도메인 문서 공간을 읽을 때는 자신의 작업기준 정보에 맞는 것을 최우선으로 확인한다.


## 문서 관리 원칙

각 문서에는 하나의 명확한 목적만 담는다.
AI Agent가 일반적인 지식만으로 처리할 수 있는 내용은 별도 문서로 만들지 않는다. 예를 들어 일반적인 파일 검색 방법, Spring의 기본 사용법, 수시로 변하는 전체 파일 목록 등은 지침 문서로 관리하지 않는다.
프로젝트 구조나 정책이 변경되면 관련 문서의 본문과 파일명을 함께 검토하고 갱신한다.
파일명은 AI Agent가 내용을 열어보지 않고도 읽어야 할 문서인지 판단할 수 있도록 구체적인 한글 이름으로 작성한다.
