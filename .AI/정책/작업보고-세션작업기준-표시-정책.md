# 작업보고 세션작업기준 표시 정책

이 문서는 AI Agent가 현재 세션에서 확정된 사용자 역할과 업무범위를 잊지 않도록, 최종 답변의 시작 형식을 고정한다.
최종 답변에는 다음 응답이 포함된다.

- 조사, 분석, 설계, 제안 결과 안내
- 코드 또는 문서 변경 결과 안내
- 검증 결과 및 완료 보고
- 사용자의 질문에 대한 최종 답변
작업 착수 또는 진행 상황 안내, 역할과 업무범위를 확인하기 위한 질문, 단순 선택지 출력, 선택 오류 안내에는 작업 기준 표시를 붙이지 않는다.

## 적용 시점

`작업시작-사용자역할과-업무범위-확인-정책.md`에 따라 역할과 업무범위 확인이 완료된 뒤부터 적용한다.
역할과 업무범위 확인이 끝나기 전에는 이 정책을 적용하지 않고, 역할과 업무범위를 확인하기 위한 질문만 수행한다.

## 최종 답변 시작 형식

역할과 업무범위 확인이 완료된 뒤 AI Agent는 세션이 종료될 때까지 최종 답변의 첫 줄에 다음 작업 기준 표시를 적는다.

```text
[현재 세션은 {역할명}, {업무범위명} 관점에서 답변합니다]
```

AI Agent의 최종 답변에서는 색상 표시를 보장할 수 없으므로, 필요한 경우 Markdown 굵게 표시만 사용할 수 있다.
보조 스크립트가 CLI에 출력하는 작업 기준 표시는 ANSI 색상을 지원하며, 기본값은 터미널 지원 여부에 따라 자동 적용된다.
`Framework 개발` 역할은 업무범위를 선택하지 않으므로 업무범위명은 `전체 영역`으로 표시한다.

```text
[현재 세션은 Framework 개발, 전체 영역 관점에서 답변합니다]
```

비즈영역 역할은 선택한 업무 도메인과, 실제 존재하는 같은 회사의 공통 영역을 함께 표시할 수 있다.

```text
[현재 세션은 비즈영역 개발, 업무 도메인 com.{예시회사명}._{예시업무명} 및 회사 공통 com.{예시회사명}.__{예시회사명}Common 관점에서 답변합니다]
```

## 작업보고 세션작업기준 표시 정책 목적

작업 기준 표시는 현재 세션 컨텍스트 유지를 위한 기준 문장이다.
AI Agent는 작업 중 현재 역할과 업무범위가 불명확해지면 추정해서 진행하지 않고, 사용자에게 역할과 업무범위를 다시 확인한다.
새 세션이 시작되면 이전 세션의 작업 기준 표시를 신뢰하지 않고 `작업시작-사용자역할과-업무범위-확인-정책.md`에 따라 다시 확인한다.
