# 작업시작 사용자역할과 업무범위 확인 정책

이 문서는 AI Agent 새 세션이 열렸을때 모든 대화와 작업에 앞서 적용해야 하는 최우선 지침이다.


## 기본 원칙

새 세션이 시작되면 먼저 사용자의 역할과 업무범위를 확인한다.
확인이 완료될 때까지 역할과 업무범위를 확인하기 위한 질문 외에는 답변, 조사, 분석, 코드·문서 수정, 명령 실행 등 어떠한 작업도 절대 진행하지 않는다.
역할과 업무범위 확인 문구는 가능하면 직접 작성하지 않고 보조 스크립트의 출력을 그대로 사용한다. 보조 스크립트를 사용할 수 없는 경우에만 이 문서의 원칙에 맞춰 직접 확인한다.


## 확인 절차

역할 질문은 다음 명령의 출력으로 진행한다.

```powershell
python .AI/_ai-generated/scripts/session/confirm-role-and-work-area.py --color auto role-question
```

사용자가 역할 번호를 선택하면 다음 명령으로 선택 결과를 확인한다.

```powershell
python .AI/_ai-generated/scripts/session/confirm-role-and-work-area.py --color auto role-selected {roleNumber}
```

`Framework 개발` 역할은 별도 업무범위를 선택하지 않는다.

`Framework 개발`이 아닌 역할은 다음 명령으로 현재 저장소의 실제 패키지 구조에서 업무 도메인 선택지를 생성한다. 회사 공통 영역은 직접 선택지로 제공하지 않고, 선택한 업무 도메인과 같은 회사의 수정 가능 범위로만 안내한다.

```powershell
python .AI/_ai-generated/scripts/session/confirm-role-and-work-area.py --color auto area-question
```

사용자가 업무범위 번호를 선택하면 다음 명령으로 선택 결과를 확인한다.

```powershell
python .AI/_ai-generated/scripts/session/confirm-role-and-work-area.py --color auto area-selected {roleNumber} {areaNumber}
```

`새 작업영역 추가`가 선택되면 아직 업무 영역을 생성하지 않고, `companyName.domainName` 형식의 신규 업무 영역 식별자를 먼저 입력받는다. 입력값을 받은 뒤에는 `.AI/작업-수행절차/신규-업무영역-추가-절차.md`를 따른다.


## 세션 작업 기준 유지
사용자의 역할과 업무범위가 확인되면 다음 형식으로 출력하여 Agent의 답변이 해당 역할과 업무범위를 메인 관점으로 두고 있음을 밝힌다.

```text
[현재 세션은 {역할명}, {업무범위명} 관점에서 답변합니다]
```

`Framework 개발` 역할의 업무범위명은 `전체 영역`으로 표시한다.
비즈영역 역할에서 선택한 업무 도메인과 같은 회사의 공통 영역 패키지가 실제로 존재하면 업무범위명에 함께 표시한다.

```text
[현재 세션은 비즈영역 개발, 업무 도메인 com.{예시회사명}._{예시업무명} 및 회사 공통 com.{예시회사명}.__{예시회사명}Common 관점에서 답변합니다]
```

같은 회사의 공통 영역 패키지가 실제로 없으면 선택한 업무 도메인만 표시한다.
AI Agent는 해당 세션이 종료될 때까지 최종 답변의 첫 줄에 현재 세션 작업 기준을 표시하여 자신이 대답하고 작업해야 하는 영역을 잊지 않도록 한다.
작업 기준 표시는 조사, 분석, 설계, 제안, 변경 결과, 검증 결과, 완료 보고, 사용자의 질문에 대한 최종 답변에만 적용한다.
작업 착수 또는 진행 상황 안내, 역할과 업무범위를 확인하기 위한 질문, 단순 선택지 출력, 선택 오류 안내에는 작업 기준 표시를 붙이지 않는다.
AI Agent의 최종 답변에서는 색상 표시를 보장할 수 없으므로, 필요한 경우 Markdown 굵게 표시만 사용할 수 있다.
작업 중 현재 역할과 업무범위가 불명확해지면 추정해서 진행하지 않고, 사용자에게 역할과 업무범위를 다시 확인한다.
새 세션이 시작되면 이전 세션의 작업 기준 표시를 신뢰하지 않고 이 정책에 따라 다시 확인한다.


## 작업을 위한 코드 참고 순서

기존 구현 코드의 참조가 필요한 경우 아래의 순서로 진행한다.
1. Framework 영역
2. `_webFrameworkExample`
3. 자신의 작업 기준 영역 (같은 회사의 공통 영역 포함)
4. 다른 도메인 영역을 참고할 수 있지만, `Framework 개발` 역할이 아닌 경우 꼭 필요한 시점에만 참고 한다.


## 세션 작업 기준 별 작업 범위

문서 및 코드 수정이 아닌 질문에 대한 답변은 현재의 작업기준과 상관 없이 답변 할수 있다.
사용자가 별도로 명시하지 않는 이상 작업 요청 사항은 현재의 작업기준 영역에서의 작업으로 간주한다.
문서 및 코드 수정과 관련된 요청의 경우는 아래 룰을 반드시 따라야 한다
- `Framework 개발`은 Framework, 회사 공통 및 모든 업무 도메인 영역을 자유롭게 조사하고 수정할 수 있다.
- `비즈영역 개발`은 선택한 업무 도메인과 같은 회사의 공통 영역을 수정할 수 있다. Framework 또는 다른 도메인 수정은 사유와 영향 범위를 설명하고 사용자 확인 후 수행한다.
- `비즈영역 설계` 및 `비즈영역 기획`은 저장소의 모든 영역과 관련해 질문과 답변을 받을 수 있지만 AI Agent는 어떠한 경우에도 작업자의 요청에 대해 실제 코드 영역을 수정해서는 안된다.
