# 프로젝트 코드영역 구분

이 문서는 새 코드를 어디에 둘지, 기존 코드가 어떤 성격의 영역에 속하는지 판단할 때 본다.
전체 프로젝트 설명은 루트 `AI.md`를 기준으로 하고, 여기서는 코드 배치 경계만 다룬다.

## 기본 영역

- `src/main/java/com/_sptek/__webFramework`: 여러 업무 프로젝트가 재사용하는 Framework코드영역
- `src/main/java/com/_sptek/_webFrameworkExample`: Framework코드영역 사용 방식을 보여주는 예시 업무 프로젝트 영역
- `src/main/java/com/{companyName}/__{companyName}Common`: 실제 업무 프로젝트 사이에서 공유할 코드가 있을 때 쓰는 영역
- `src/main/java/com/{companyName}/{domain}`: 실제 업무 기능을 구현하는 영역

`{companyName}`은 실제 업무 프로젝트의 최상위 패키지명이다. 한 저장소 안에 하나 이상 존재할 수 있으므로 특정 이름으로 고정해서 설명하지 않는다.

## 배치 기준

Framework코드영역에는 여러 업무 프로젝트에서 반복해서 사용할 수 있는 기술 기반 코드를 둔다.
특정 업무 규칙, 화면 정책, 업무 데이터 모델은 실제 업무 프로젝트 영역에 둔다.

업무 프로젝트 사이에서 공유할 필요가 있는 코드는 `com.{companyName}.__{companyName}Common`을 검토한다.
다만 프레임워크 핵심 코드나 특정 도메인 전용 로직을 이 영역에 섞지 않는다.
