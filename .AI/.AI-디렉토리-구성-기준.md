`.AI/`는 특정 Agent에 종속되지 않는 공통 작업 기준을 보관한다. 내부 문서는 다음 기준으로 구분한다.

### `policies`

Agent가 작업 중 반드시 지켜야 하는 규칙을 둔다.
사용자 변경 보존, 민감 파일 처리, Base 프레임워크 변경 경계, 커밋 기준처럼 작업자의 행동을 제한하거나 판단 우선순위를 정하는 내용이 여기에 속한다.

### `context`

프로젝트를 이해하기 위한 현재 구조와 사실 관계를 둔다.
저장소 구조, 프레임워크와 예시 업무 프로젝트의 경계, 리소스 구조, 런타임 진입점, profile 구성처럼 “이 프로젝트가 현재 어떻게 구성되어 있는가”를 설명한다.

### `procedures`

반복 작업을 수행하는 절차를 둔다.
코드 분석, 코드 변경, 커밋 전 점검, 읽기 전용 DB 조회, tree-content export/import처럼 “특정 작업을 어떤 순서로 진행할 것인가”를 설명한다.

### `assets`

절차에서 재사용하는 실행 파일과 자료를 둔다.
스크립트, SQL snippet, 문서 template처럼 설명 문서가 아니라 실제 작업 중 실행하 거나 참조하는 보조 자산이 여기에 속한다.

구분이 애매하면 문장의 성격을 기준으로 판단한다.
“반드시 지켜야 한다”는 규칙이면 `policies`, “현재 구조는 이렇다”는 설명이면 `context`, “이 순서로 진행한다”는 절차이면 `procedures`, “실행하거나 복사해서 쓴다”는 파일이면 `assets`에 둔다.
